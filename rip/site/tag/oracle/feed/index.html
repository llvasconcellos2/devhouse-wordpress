oracle – DevHouse Internet Software Development House http://localhost:8080 Desenvolvimento de Sistemas para Internet e Mobile Tue, 23 Jun 2026 08:58:43 +0000 pt-BR hourly 1 https://wordpress.org/?v=4.8.14 Root no Motorola Droid Razr XT910 [Jelly Bean 4.1.2] http://localhost:8080/root-no-motorola-droid-razr-xt910-jelly-bean-4-1-2/ http://localhost:8080/root-no-motorola-droid-razr-xt910-jelly-bean-4-1-2/#comments Wed, 08 Jan 2014 04:12:09 +0000 http://www.devhouse.com.br/?p=619 Para executar o exploit que dá acesso ao usuário root no Android do Motorola Droid Razr XT910 você vai precisar dos seguintes arquivos / programas:

  • Drivers da Motorola (Motorola Device Manager)
  • Oracle Virtualbox (Versão 4.3.6 para Windows)
  • Appliance do Virtualbox contendo o root exploit

Feito os downloads vamos iniciar o procedimento:

  1. Instale os Motorola Device Manager.
  2. Instale o VirtualBox.
  3. Descompacte o arquivo Root_Bionic_JB.rar.
  4. Abra o VirtualBox e vá em Arquivo – Importar Applience (Crtl+I).
  5. Selecione o arquivo Root_Bionic_JB.ova e clique em Próximo (N).
  6. Não altere nada nas configurações do Appliance. Apenas clique em Importar.
  7. Depois vá em Configurações – USB remova os dispositivos listados e adicione o seu Razr (Motorola XT910) e depois clique em OK conforme a imagem abaixo:

    Selecionando o Razr na lista de dispositivos USB da VM

  8. Inicie a VM clicando em Iniciar (T).
  9. O Razr e o seu computador devem estar na mesma rede WIFI. A VM vai pausar exibindo uma mensagem pedindo para você conectar o Razr ao computador: No Razr abra o app “Arquivos”, selecione Armazenamentos remotos – Adicionar Armazenamento.
  10. No campo Endereço de IP do host informe o “Host IP address” exibido na VM. No meu caso: 192.168.0.13.
  11. Nome do domínio: MYGROUP
  12. Nome da pasta compartilhada: share
  13. Nome do usuário: guest
  14. Senha: bionic
  15. Clique em conectar no Razr e depois Enter na VM.
  16. A VM vai exibir a mensagem: CONFIRMED! Phone connected to SMB. Caso essa mensagem não seja exibida verifique as informações, usuário/senha e tente novamente.
  17. Verifique se o seu aparelho está com a Depuração USB habilitada (Configurar – Opções do Programador – Depuração USB).
  18. Verifique se o seu aparelho está configurado para utilizar o protocolo MTP (Configurar – Armazenamento – Botão Menu – Conexão USB computador – Dispositivo de mídia (MTP)).
  19. Aperte Enter na VM.
  20. O processo de exploit iniciará. O seu celular pode parecer que reiniciou, ignore.
  21. Aguarde até que o telefone vibre então pressione Enter na VM.
  22. Ele vai exibir a mensagem: “Cleaning Up” e depois vai pedir para teclar Enter. Pressione Enter na VM.
  23. Seu telefone vai reiniciar e seu Razr já estará rootado.

Este é o primeiro Post de múitos que devem vir, ou pelo menos assim espero!

Resolvi escrever este blog para registrar as dificuldades e soluções que encontro no dia a dia envolvendo tecnologia. Para ser preciso, resolvi mesmo escrever para registrar as minhas aventuras no mundo Linux.

Conheço Linux desde 1999 quando fiz um curso do Conectiva Linux (hoje Mandriva) na Dualline, onde trabalhei na época. Inclusive meu professor foi o Augusto Campos, mais conhecido como Brain do site br-linux.org. Depois do curso administrei um servidor web com Linux Slackware na SCTEL uma empresa de telecomunicações que trabalhei depois que sai da Dualline e além disso já fiz várias instalações de servidores como prestador de serviços de infra-estrutura de rede. Normalmente eu instalava o RedHat ou o CentOS depois que foi lançado. Apesar de ter contato com Linux durante toda a minha vida profissional até 2011 não tinha usado em minha máquina pessoal. Passei a usar o Ubuntu como meu sistema principal em uma máquina dual boot com o Windows 7 quando fui trabalhar com desenvolvimento web Java na Totvs (sim, eu já trabalhei em quinhentas empresas diferentes e isso é assunto para um outro post). Tomei esta decisão após constatar que os servidores de aplicação Java como JBoss, da RedHat, ou o GlassFish, da Oracle, se integravam melhor ao Linux do que ao Windows. Aliás é comum um servidor Java Enterprise rodar no Linux.

Então, minha experiência diária com o Linux foi com o Ubuntu logo após da adoção do Unity como interface gráfica padrão. Não sei se foi culpa minha, do Unity, dos milhões de tweeks que fiz no sistema mas a minha relação com o Ubuntu sempre esteve longe de ser estável. Por isso agora resolvi usar o Linux Mint Debian pois ouvi dizer que é mais estável e um pouco mais amigável do que o Debian mesmo. Então o que me motivou a escrever este blog é que toda a vez que formato minha máquina para instalar uma distro diferente do Linux faço vários ajustes no sistema e instalo vários programas e depois esqueço como se faz :-P. Desta vez resolvi anotar!

Também posso compartilhar minhas experiências com programação Web e Mobile e algumas notícias sobre assuntos pertinentes ou qualquer outra coisa que vier a cabeça.

Espero que gostem do blog para ele servir para algo mais do que apenas minhas anotações.

Abraços,

Leonardo Lima de Vasconcellos
