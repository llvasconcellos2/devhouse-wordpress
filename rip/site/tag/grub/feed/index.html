Um notebook feito para ser leve, portátil e com bateria durável. E ele realmente é. No geral é um notebook bom considerando o seu processador Core 2 Duo de 1,3 GHz de baixo consumo de energia.

As teclas de atalho padrão neste notebook para o funcionamento do controle de brilho são fn (function) + seta para cima ou seta para baixo. Logo após a instalação do Linux Mint 12 Debian Edition eu tentei ajustar o brilho e nada. O problema é igual para todas as distribuições que testei: Ubuntu 11.10, Ubuntu 12.04 e Fedora 16. Ao teclar fn + seta para cima aparecia o ícone de brilho na tela mas nada acontecia e ao teclar fn + seta para baixo o brilho era ajustado ao máximo e o ícone mostrava que estava um pouco abaixo do máximo, o que era estranho.

Pesquisando na net encontrei a solução do problema e parece ser comum entre notebooks com chipset Intel GS45 + ICH9MS, por isso se o seu notebook posui este chipset ou se está tendo o mesmo problema tente a correção a seguir:

A correção consiste em adicionar um parâmetro de configuração do Kernel do Linux no boot. Para testá-lo tecle “e” na tela de boot do linux para editar os comandos de boot e insira acpi_osi= no final da linha “linux /boot/…”. O do meu computador ficou assim (linha 9):

setparams 'LinuxMint GNU/Linux, with Linux 3.3.0-6.dmz.1-liquorix-amd64

insmod gzio
insmod part_msdos
insmod ext2
set root='(/dev/sda, msdos1) '
search --no-floppy --fs-uuid --set=root 321e9577-ad48-4370-9276-06399e26b6da
echo 'Loading Linux 3.3.0-6.dmz.1-liquorix-amd64 ... '
linux /boot/vmlinuz-3.3.0-6.dmz.1-liquoriz-amd64 root=UUID=321e9577-ad48-4370-9276-06399e26b6da ro quiet acpi_osi=
echo 'Loading initial ramdisk ...'
initrd /boot/initrd.img-3.3.0-6.dmz.1-liquorix-amd64

Segundo informações, o kernel do Linux por padrão se idenditifica como sendo qualquer versão do Windows para a BIOS e o problema é que algumas BIOS tem desvios de código para resolver bugs do Windows Vista e por isso acaba impactando no Linux.

Outra alternativa é usar o parâmetro abaixo caso a sua BIOS tiver suporte ao Linux:

acpi_osi=”Linux”

Ou usar o seguinte parâmetro para fazer com que o Linux se identifique como Windows Vista e executar comandos de compatibilidade com o mesmo:

acpi_osi=”Windows 2006″

Para usar o parâmetro de forma permanente

Conforme dito acima, para testar o parâmetro você pode teclar “e” no menu de boot do Grub2 e inserir o parâmetro, mas isso só vai funcionar até o próximo boot. Para deixar o parâmetro de forma permanente você precisa editar um arquivo de configuração do grub conforme asseguir:

$ sudo nano /etc/default/grub

O editor nano vai abrir o arquivo e seu conteúdo será parecido com esse:

# If you change this file, run 'update-grub' afterwards to update
 # /boot/grub/grub.cfg.
 # For full documentation of the options in this file, see:
 #   info -f grub -n 'Simple configuration'

GRUB_DEFAULT=0
GRUB_TIMEOUT=5
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`
GRUB_CMDLINE_LINUX_DEFAULT="quiet"
GRUB_CMDLINE_LINUX=""

# Uncomment to enable BadRAM filtering, modify to suit your needs
# This works with Linux (no patch required) and with any kernel that obtains
# the memory map information from GRUB (GNU Mach, kernel of FreeBSD ...)
#GRUB_BADRAM="0x01234567,0xfefefefe,0x89abcdef,0xefefefef"

# Uncomment to disable graphical terminal (grub-pc only)
#GRUB_TERMINAL=console

# The resolution used on graphical terminal
# note that you can use only modes which your graphic card supports via VBE
# you can see them in real GRUB with the command `vbeinfo'
#GRUB_GFXMODE=640x480

# Uncomment if you don't want GRUB to pass "root=UUID=xxx" parameter to Linux
#GRUB_DISABLE_LINUX_UUID=true

# Uncomment to disable generation of recovery mode menu entries
#GRUB_DISABLE_RECOVERY="true"

# Uncomment to get a beep at grub start
#GRUB_INIT_TUNE="480 440 1"

Adicione o parâmetro na linha GRUB_CMDLINE_LINUX_DEFAULT conforme abaixo:

GRUB_CMDLINE_LINUX_DEFAULT="quiet acpi_osi="

Se você precisar usar parâmetros que contém aspas (“) você precisa fazer o escape das aspas conforme abaixo:

GRUB_CMDLINE_LINUX_DEFAULT="quiet acpi_osi=\"Linux\""

Salve o arquivo e entre com o comando a seguir para atualizar as configurações do grub:

$ sudo update-grub

That’s all folks!

]]> http://localhost:8080/corrigindo-o-controle-de-brilho-no-notebook-com-linux/feed/ 28 Passo a passo da instalação do Linux Mint Debian Edition LMDE 201204 http://localhost:8080/instalacao-do-linux-mint-debian-edition-201204/ http://localhost:8080/instalacao-do-linux-mint-debian-edition-201204/#comments Tue, 29 May 2012 04:47:26 +0000 http://www.devhouse.com.br/?p=46 Neste post vamos mostrar passo a passo a instalação do Linux Mint Debian Edition que é um pouco mais simples do que a instalação do Debian Testing “Wheezy”, distribuição a qual é baseado. Para concluir este tutorial você vai precisar acesso a internet, um cd virgem, regravável ou um pendrive e um pouco de paciência.

Então, você que quer deixar o windows de lado e usar um sistema operacional de verdade ou você que já usa Linux mas está de saco cheio com o Ubuntu que tal Linux de Menta?

Vamos lá?

Instalação do Linux Mint Debian Edition 201204


O primeiro passo é fazer o donwload da imagem ISO do cd de instalação do LMDE. Para isso vá ao site www.linuxmint.com, selecione download e Linux Mint Debian ou clique aqui para ir direto. Em important links selecione MATE/Cinnamon para Gnome 2 e 3 respectivamente ou Xfce para o Desktop Xfce. Existem opções de 32 ou 64 bits. No meu caso escolhi Gnome e 64 bits.

Download do Linux Mint Debian Edition.

Na próxima página você pode escolher um mirror mais próximo para fazer download via http ou torrent. Eu escolhi torrent para garantir a integridade do download já que torrent verifica md5 do arquivo.

Escolha um mirror para download.

Com o arquivo iso em mãos você pode gravar um cd ou gerar um pendrive de boot. O meu computador não tem drive ótico então vou precisar do Unetbootin. Um utilitário para gerar pendrives de boot para instalar Linux. Existem versões para windows, linux e mac.

Download do Unetbootin, gerador de pendrive de boot.

Ao executar o Unetbootin selecione “Imagem de disco” ISO e no botão “…” navege até o arquivo que você fez download do LMDE. Selecione a letra da unidade onde está inserido o pendrive e clique em OK.

É importante já ter o pendrive formatado em Fat32 antes para que este passo seja executado com sucesso.

Preparando o pendrive para instalação.

Após a criação do disco de boot, insira o mesmo no seu computador, entre na bios e selecione a opção de ordem de boot para o pendrive ou cd correto. Este passo é importante e depende do modelo de cada BIOS e por isso não é descrito aqui.

Ao ligar o computador com o disco de boot inserido aparecerá a seguinte tela. Selecione default e pressione enter:

Menu de boot do Grub com UnetBootin


O sistema fará boot no livecd e quando estiver pronto clique no ícone Install Linux Mint

Instalar Linux Mint Debian Edition


Escolha a sua lingua. Eu prefiro o Linux em inglês para facilitar a busca de informações na internet mas você pode escolher a lingua que preferir.

Seleção de línguas para instalação


Escolha seu fuso horário:

Escolha seu fuso horário.


Escolha o layout do seu teclado. Teclados com cedilha são português (Brazil) e sem ccedilha você pode escolher: Estados Unidos –  English International.

Escolha o layout do teclado.

Selecione o HD a ser instalado o sistema.

Escolha o HD para instalação

Selecione a partição onde sera instalado o Linux. No meu caso o disco está limpo e por isso as partições deverão ser criadas. Clique em Edit partitions.

Seleção da partição para instalação.

A tela do Gparted abrirá. Clique em Partition -> New.

Criação da nova partição.

No campo Free space following (MiB) preencha o tamanho em megabytes que vai sobrar no disco após a criação da primeira partição. Basicamente você deixa o espaço livre que você usará para a partição de swap. No meu caso vou usar o mesmo valor que tenho de memória. Provavelmente não tem necessidade de tanto mas na dúvida mal não vai fazer. Em File system selecione ext4 e clique em Add.

Deixando espaço para a partição de swap.

Selecione o espaço unallocated, vá no menu Partition -> New. Vamos usar o restante para a partição de swap. Em File system selecione linux-swap. Clique em Add.

Criando a partição de Swap.

Seu disco ficará parecido com a figura abaixo: Vá no menu Edit -> Apply All Operations.

Resultado do particionamento.

Confirme clicando em apply.

Confirmação das alterações.

As partições serão criadas. Feche a janela do Gparted e clique no botão refresh. As partições criadas vão aparecer no instalador do linux. Clique com o botão direito do mouse na partição do tipo ext4 e selecione Assign to /. O mout point da partição primaria ficara em /. Clique em forward.

Definindo o ponto de montagem.

Entre com as informações pedidas e o nome do computador na rede (hostname).

Nome de usuário, senha e hostname.

Na próxima tela deixe o padrão ou escolha uma partição específica para o Grub.

Onde gravar o Grub?

Revise as configurações e clique em install.

Revisão das configurações.

O processo de instalação e cópia dos arquivos será iniciado. Vá tomar café e volte daqui a alguns minutos.

Cópia dos arquivos.

Ao termino da instalação clique em Quit e reinicie o computador. Não esqueça de tirar o pendrive.

Pronto!
