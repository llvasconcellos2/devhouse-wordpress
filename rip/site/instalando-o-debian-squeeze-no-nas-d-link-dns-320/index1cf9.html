Instalando o Debian Squeeze no NAS D-Link DNS-320

Por Leonardo Lima de Vasconcellos Em 21 out. 2012 Em Debian, Hardware, Linux Possui 102 Comentários

Esta é a segunda parte do guia de configuração do Debian Linux para o NAS D-Link DNS-320. Estou considerando que você já possui uma instalação funcionando do fonz fun_plug no seu NAS, caso contrário consulte a primeira parte deste guia:

  • Instalando o fonz fun_plug no D-Link DNS-320 NAS

    outubro 21. 201296Comment

    O NAS ou Network Attached Storage DNS-320 da D-Link é um servidor de arquivos muito bom e barato que me ajuda a sincronizar...

Primeiro Passo: Download do Debian Squeeze

Já existe um pacote do Debian Squeeze preparado para o NAS. Faça o download neste link:

ftp://ftp.dlink.pl/dns/dns-320/driver_software/DNS-320&325_linuxDebian_revA_Squeeze_all_en_20110111.zip

Aproveitei e criei um mirror para o caso deste link sair do ar:

http://www.4shared.com/zip/NzsLLdDt/DNS-320325_linuxDebian_revA_Sq.html

Após o download, descompacte o arquivo. Dentro estão 3 arquivos:

total 154324
-rw-r--r-- 1      2254 Jan 11  2011 DNS-320&325_howto_Linux-Debian-in-chroot.txt
-rw-r--r-- 1       569 Sep 21  2010 fun_plug.debian
-rw-r--r-- 1 158011675 Sep 22  2010 squeeze.tar.gz

Segundo Passo: Instalando o Debian Squeeze

Copie os arquivos fun_plug.debian e squeeze.tar.gz para o Volume_1 no NAS. Renomeie o arquivo fun_plug que já estava no NAS para fun_plug.old e renomeie o arquivo fun_plug.debian para fun_plug.

Não reinicie o NAS ainda. Vamos precisar do fun_plug que já está rodando para configurar o Linux. Conecte via ssh ao seu NAS.

# Substitua 192.168.0.2 pelo IP do seu NAS.
ssh root@192.168.0.2

Agora execute os seguintes comandos:

cd /mnt/HD/HD_a2
gunzip squeeze.tar.gz
tar -xf squeeze.tar
rm squeeze.tar

Reinicie o NAS.

Terceiro Passo: Configurando o Debian

Conecte via ssh ao seu NAS. Usuario root e senha dlink.

# Substitua 192.168.0.2 pelo IP do seu NAS.
ssh root@192.168.0.2

Caso não consiga se conectar em rasão da chave de criptografia ter mudado apague o arquivo ~/.ssh/known_hosts e tente novamente.

Edite o arquivo /etc/apt/sources.list.

nano /etc/apt/sources.list

Remova o repositório que já se encontra listado e insira os seguintes:

deb http://ftp.debian.org/debian/ squeeze main contrib non-free
deb-src http://ftp.debian.org/debian/ squeeze main contrib non-free

deb http://backports.debian.org/debian-backports squeeze-backports main contrib non-free
deb-src http://backports.debian.org/debian-backports squeeze-backports main contrib non-free

deb http://security.debian.org/ squeeze/updates main contrib non-free
deb-src http://security.debian.org/ squeeze/updates main contrib non-free

Salve o arquivo e execute os comandos:

apt-get update

# Instalar sudo
apt-get install sudo

# Instalar locales
apt-get install locales

# Selecione a sua locale. Eu uso en_US.UTF-8
dpkg-reconfigure locales

# Atualizar o sistema
apt-get upgrade

# Alterar a senha de root
passwd

# Configure seu Timezone
dpkg-reconfigure tzdata

# Adicionar um usuário para não usar o root.
adduser leonardo
adduser leonardo sudo

Pronto! Agora vocé tem um Debian completo rodando no seu NAS.
